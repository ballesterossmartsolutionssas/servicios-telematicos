==================================================
PASO 1 - MOSTRAR CONFIGURACION EN CLIENTE
==================================================

En terminal CLIENTE ejecuta:
hostnamectl
ip -brief addr
sudo cat /etc/systemd/resolved.conf
resolvectl status | sed -n '1,80p'

QUE DECIR
- Aqui muestro la configuracion del resolvedor del sistema.
- DNSOverTLS=yes indica que las consultas DNS deben ir cifradas.
- Los servidores configurados son Cloudflare y Google con nombre de autenticacion.

QUE SEÑALAR
- En resolved.conf:
  DNS=1.1.1.1#cloudflare-dns.com 8.8.8.8#dns.google
  FallbackDNS=...
  DNSOverTLS=yes
  Domains=~.

- En resolvectl status:
  +DNSOverTLS
  Current DNS Server
  DNS Servers

QUE SIGNIFICA CADA COSA
- systemd-resolved: servicio de resolucion DNS de Ubuntu.
- DNSOverTLS=yes: obliga a usar TLS para consultas DNS.
- 853/tcp: puerto estandar de DNS over TLS.
- 53: puerto clasico de DNS sin cifrar.

==================================================
CONCEPTOS CLAVE QUE TE PUEDEN PREGUNTAR
==================================================

QUE ES DNS OVER TLS
- Es un mecanismo para cifrar consultas DNS usando TLS.
- Protege la privacidad de los dominios consultados.

QUE PROBLEMA RESUELVE
- Evita que un tercero vea facilmente que dominios estas consultando cuando captura trafico.

QUE PUERTO USA
- Usa el puerto 853 TCP.

QUE PASA EN DNS NORMAL
- Las consultas viajan sin cifrar por el puerto 53.
- El dominio consultado puede verse en texto claro.

DIFERENCIA ENTRE 853 Y 53
- 853: DNS sobre TLS, cifrado.
- 53: DNS clasico, normalmente sin cifrar.

QUE DEMUESTRA resolvectl
- Que el cliente esta configurado para usar DoT.
- Que las consultas estan funcionando.
- Que el transporte es local o cifrado.
